# Week14 Day1 数据分析记录

## 数据来源

- 数据库：UniProtKB / Swiss-Prot
- 物种：Human, organism_id:9606
- 审核状态：reviewed:true
- 蛋白长度范围：50-500 aa
- 标签类型：GO Molecular Function

## 原始数据统计

- 有效蛋白样本数：1783
- GO标签出现次数总数：6652
- GO标签去重后总数：1701
- 平均每个蛋白GO标签数：3.73

## Top-50标签空间

- 标签空间大小：50
- Top-50标签出现次数总和：2716
- Top-50标签出现次数覆盖率：40.8%
- 过滤后剩余样本数：1213 / 1783
- 过滤后样本覆盖率：68.0%
- 过滤后平均每样本标签数：2.25

## 结论

本周正式采用 Top-K = 50 作为多标签分类任务的标签空间。

选择理由：

1. Top-50过滤后仍保留1213个样本，足够启动小规模多标签训练。
2. 过滤后平均每个样本有2.25个标签，任务仍然保持多标签属性。
3. Top-50尾部标签频次约为21次，已经接近小样本边界；继续扩大K会显著增加低频标签、类别不平衡和验证集指标波动。
4. 当前阶段目标是建立稳定、可复现的多标签训练管线，而非完整覆盖所有GO term。

## Day1反思

### 1. 丢弃低频GO term的影响

丢弃低频term可以降低标签空间维度、缓解类别不平衡、提高训练稳定性；但会牺牲罕见功能的预测能力，并使任务偏向常见功能类别。本阶段这是合理的工程折中。

### 2. 平均标签数是否符合预期

原始平均标签数为3.73，Top-50过滤后为2.25，符合蛋白功能多样性的预期。蛋白功能通常不是互斥类别，而是多个功能属性的组合，因此必须使用多标签分类建模。

### 3. 数据源更换的设计原则

如果未来从UniProt切换到Ensembl，只需重写数据拉取层，保证最终输出schema仍为：

```python
["Entry", "sequence", "go_list"]
```

下游标签分析、Dataset、训练与测试模块无需修改。这体现了接口隔离和单一职责原则。
