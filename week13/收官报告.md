## 二、Day-by-Day 关键决策与产出

### Day 1-2：确定性基座建立
- 发现历史 Baseline 结果存在随机性抖动，定位到 `set_seed` 规范不完整（缺少 `cudnn.deterministic=True` 等关键项）。
- 建立标准化 `set_seed()` 函数，并在 **Baseline v5** 上验证：跨脚本运行 ΔAcc = 0.0000，彻底消除历史噪声。
- 产出：`week13/day1/baseline_checkpoint_v5.pt`（字典包裹，key=`model_state_dict`）。

### Day 3：数据增强探索
- 实现 `AugmentedProteinDataset`，支持"随机替换"与"随机裁剪"两种增强策略，通过 `augment` 标志位区分训练/推理。
- **关键发现**：裁剪增强（随机截断序列片段）对本任务是**负增益**——蛋白质定位信号往往依赖信号肽等局部关键片段，随机裁剪会破坏这些生物学信号。
- **保留策略**：仅使用保守的随机替换（`use_substitution=True, use_crop=False`），最终验证为唯一带来正增益的增强手段（Test F1 = 0.4806，优于当时 Baseline）。

### Day 5：类别不平衡加权
- 实现两种加权策略：`inverse`（标准 balanced 权重）与 `sqrt_inverse`（平方根平滑倒频率）。
- 平方根倒频率权重公式：$$w_i = \sqrt{\frac{N}{K \cdot count_i}}$$，相比标准倒频率更温和，避免稀有类权重过度放大导致训练不稳定。
- 结果：加权模型（sqrt）在牺牲部分整体 Accuracy（0.5997）的代价下，Macro F1 提升至 0.4791，稀有类召回率有改善。
- 产出：`week13/day5/checkpoints/sqrt_inverse.pth`（裸 state_dict）。

### Day 6：正则化与 Early Stopping
- 在 Dropout 系数与 Label Smoothing 两个维度做消融，单种子下发现：
  - **配置A**：Dropout=0.1（无LS） → Macro F1 = 0.4890（**当时最优**）
  - **配置B**：Dropout=0.1 + LabelSmoothing0.1 → Test Acc 更高，但 F1 略低
- 产出：`week13/day6/day6_best_checkpoint.pt`（字典包裹，对应配置A）。

### Day 6 补充：多种子稳健性验证
- 针对 A/B 两个候选配置，各独立跑 3 个种子（42 / 114514 / 1919810），验证结论是否稳健：

  | 配置 | Macro F1 (mean ± std, n=3) |
  | :--- | :--- |
  | A. Dropout=0.1（无LS） | **0.4901 ± 0.0165** |
  | B. +LabelSmoothing0.1 | 0.4480 ± 0.0305 |

- **严谨性说明**：均值差（0.0421）小于两组 std 之和（0.0470），按"均值差 vs std之和"这一保守判断法未达到严格显著性阈值。但考虑到 n=3 时该判断法本身偏保守，且均值方向一致地支持排除 LabelSmoothing，最终结论是：**排除 LS 的方向性证据可信，但置信区间较宽**，这一点已如实记录，不做过度断言。
- 顺带完成第4次 checkpoint 交叉验证：seed=42 下 A组F1(0.4890) 确实高于B组(0.4827)，证明 `day6_best_checkpoint.pt` 保存的模型无误。

### Day 7：集成与收官

**Phase 1 - Soft Voting 集成**：将 Baseline v5、加权模型(sqrt)、正则化模型三者的 softmax 输出等权（1/3）平均。

**Phase 2 - "全部改进"综合验证**：尝试把数据增强(仅替换) + WCE(sqrt) + Dropout0.1 + EarlyStopping 四个改进叠加进同一个模型，3种子验证。

## 四、核心结论

### 结论1：集成多个"各有专精"的模型，优于把所有改进硬塞进单一模型

这是本周最具价值的发现。"全部改进"模型（F1=0.4782）反而**低于**仅做单一改进的正则化模型（F1=0.4901），差距达 −0.0119。

**原因分析**：`WCE(sqrt)` 加权和数据增强都会向训练过程引入额外的梯度噪声/方差——前者放大稀有类样本对 loss 的贡献，后者对输入序列做随机扰动。当两种噪声源同时作用于同一个模型的同一次训练时，会相互干扰，导致优化路径更不稳定（对比训练日志：全部改进组 Loss 从 1.96 降至约 1.10-1.30 区间即触发 EarlyStop，明显比单一改进组收敛更慢更颠簸）。

而 **Soft Voting 集成**恰好利用了对立的机制：三个模型**独立训练**，各自只承受一种改进/扰动，它们在测试集上的错误模式并不完全相同。集成时对 softmax 概率做平均，能够让彼此的偏差相互抵消——这正是 Ensemble 方法生效的经典原理：**模型间错误的去相关性，比单模型自身有多强更重要**。

### 结论2：数据的生物学特性应优先于通用的深度学习技巧

Day3 发现的"裁剪增强负增益"现象提示：蛋白质序列不是普通文本，随机裁剪会破坏信号肽等定位关键的局部结构信息。这提醒在生物信息学场景中，**任何通用增强/正则化手段的引入都需要结合任务的生物学先验做取舍**，而不能照搬 NLP/CV 领域的默认配置。

### 结论3：确定性工程基座是一切实验结论可信度的前提

本周通过统一 `set_seed` 规范，实现了跨脚本、跨天 **5 次独立交叉验证**均精确复现（ΔAcc/ΔF1 = 0.0000），这是所有上述结论能够被信任的根本前提。若无此基座，Day6 中 A/B 配置的细微差异完全可能被误判为"改进有效"，而实际只是随机种子噪声。

## 五、已知局限性（如实记录）

即便是最优的 Soft Voting 集成，稀有类别仍是明显的薄弱环节：

| 类别 | 训练集样本数 | 占比 | Ensemble F1 |
| :--- | :--- | :--- | :--- |
| Lysosome/Vacuole | 192 | 2.3% | **0.067**（几乎未学到） |
| Peroxisome | 93 | 1.1% | 0.250 |

这说明当前架构（无预训练迁移的 3 层 Transformer + ESM2 tokenizer）在极度稀缺样本场景下存在天花板，样本量本身的稀缺性可能比任何加权/增强技巧更根本地限制了模型表现。这一局限应被诚实呈现，而不应被集成后的整体高分掩盖。

## 六、最终工程推荐配置

> **Soft Voting 集成**
> （Baseline v5 + 加权模型sqrt + 正则化模型，等权 1/3 平均 softmax 输出）
>
> **Test Accuracy = 0.6481 | Macro F1 = 0.5117**

相比 Week13 起始的 Baseline（F1=0.4643），最终方案实现 **+0.0474（约10.2%相对提升）** 的 Macro F1 增益，且每一步改进的有效性均经过固定种子的严格验证，排除了随机性带来的虚假结论。

## 七、未来工作方向（留待后续）

1. **加权 Voting 优化**：当前 Soft Voting 为等权平均，但三个成员模型强弱不等（正则化模型 F1=0.4890 > 加权模型 F1=0.4791）。理论上给更强的成员分配更高投票权重（如 0.5:0.25:0.25）可能进一步提升集成上限，这是低成本、可快速验证的下一步实验。
2. **稀有类别的针对性方案**：考虑对 Lysosome/Peroxisome 等极稀有类别引入过采样（如 SMOTE 的序列变体）或两阶段分类（先粗分类再细分），而非依赖损失加权这一单一手段。
3. **预训练迁移的进一步利用**：当前分类头仍是从零训练的 3 层 Transformer，仅 tokenizer 复用了 ESM2，后续可探索直接微调 ESM2 编码器本体，验证真正的迁移学习收益上限。
