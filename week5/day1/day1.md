## 一、LSTM 已经很好了，问题在哪里？

回顾一下 Week 4 的 BiLSTM：

```text
输入序列：A → B → C → D → E → F → G

LSTM 的工作方式：
  读完 A，得到 h1
  读完 B，得到 h2（包含 A 的信息）
  读完 C，得到 h3（包含 A、B 的信息）
  ...
  读完 G，得到 h7（理论上包含所有信息）
```

看起来没问题？**问题在于"理论上"这三个字。**

---

## 二、LSTM 的瓶颈：信息压缩

```text
序列长度 = 100：

A → B → C → ... → Z → AA → ... → CV

LSTM 需要把前 99 个字符的信息
全部压缩进一个固定大小的向量 h（比如 64 维）

然后用这个 h 来预测第 100 个位置
```

这就像让你**只看最后一页摘要，来回答整本书的问题**。

两个具体问题：

**问题 1：远距离依赖（Long-range Dependency）**

```text
蛋白质序列例子：

位置 5 的氨基酸 K（带正电）
和
位置 87 的氨基酸 D（带负电）

它们之间有盐桥，功能上强相关。

但 LSTM 从位置 5 传递到位置 87，
中间经过了 82 次隐藏状态更新，
位置 5 的信息早就被稀释了。
```

**问题 2：顺序计算，无法并行**

```text
LSTM 必须按顺序计算：
  先算 h1，再算 h2，再算 h3...

序列长度 = 1000，就必须串行走 1000 步。
GPU 有几千个核，但 LSTM 用不上。
```

---

## 三、Attention 的核心思想

Attention 的解法非常直接：

> **不通过隐藏状态中转，直接让每个位置"看"序列中的所有其他位置。**

```text
还是上面的蛋白质例子：

计算位置 87 的表示时：
  不是靠 h86 传递过来的信息
  而是直接问：
    "位置 87 和位置 1 有多相关？" → 得到一个分数
    "位置 87 和位置 2 有多相关？" → 得到一个分数
    ...
    "位置 87 和位置 5 有多相关？" → 得到一个高分数（K-D 盐桥）
    ...
    "位置 87 和位置 86 有多相关？" → 得到一个分数

  然后按照这些分数，对所有位置的信息做加权求和
  → 得到位置 87 的新表示
```

用一句话概括：

$$\text{新表示}_i = \sum_{j=1}^{L} \alpha_{ij} \cdot \text{原始信息}_j$$

其中 $$\alpha_{ij}$$ 是位置 $$i$$ 对位置 $$j$$ 的**注意力权重**，所有权重加起来等于 1。

---

## 四、和 LSTM 的对比

| | LSTM | Attention |
|---|---|---|
| 远距离依赖 | 靠隐藏状态中转，容易衰减 | 直接计算任意两个位置的关系 |
| 并行计算 | ❌ 必须顺序计算 | ✅ 所有位置同时计算 |
| 序列长度限制 | 长序列性能下降明显 | 理论上无限制（实践中有显存限制） |
| 计算复杂度 | $$O(L)$$ | $$O(L^2)$$（每对位置都要算） |

注意最后一行：**Attention 的代价是 $$O(L^2)$$**，序列越长，计算量越大。这是它的主要缺点。

## Q1 ✅ 方向正确，补充一个细节

你说"进行类似的跳过处理"，方向对。但 Attention 的处理方式和 LSTM 不太一样：

```text
LSTM 的方式：pack，物理上跳过 padding 位置，不计算

Attention 的方式：不能物理跳过（因为要同时计算所有位置）
  → 改用"掩码（Mask）"
  → 在计算注意力分数之后，把 padding 位置的分数设为 -∞
  → 经过 softmax 后，-∞ 变成 0
  → 相当于"这个位置的权重为 0，不贡献任何信息"
```

用图来看：

```text
注意力分数矩阵（4x4，序列长度=4，但位置3是padding）：

        pos0  pos1  pos2  pos3(pad)
pos0  [  2.1   1.3   0.8   1.9  ]
pos1  [  0.5   2.4   1.1   1.7  ]
pos2  [  1.2   0.9   2.3   1.5  ]

加 mask 后（padding 列设为 -∞）：

        pos0  pos1  pos2  pos3(pad)
pos0  [  2.1   1.3   0.8   -∞   ]
pos1  [  0.5   2.4   1.1   -∞   ]
pos2  [  1.2   0.9   2.3   -∞   ]

softmax 后，-∞ 那列变成 0，padding 位置完全被忽略 ✅
```

这个操作叫 **Padding Mask**，Week 5 Day 3 会写代码实现它。

## Q2 ✅ 计算正确，分析也对

$$512^2 / 512 = 512 \text{ 倍}$$

补充一点你说的"并行"：

```text
LSTM：串行走 512 步，每步很快，但必须等上一步完成
  → 总时间 ≈ 512 × 单步时间

Attention：同时计算 512×512 = 262144 个分数
  → 但这 262144 个计算完全独立，可以全部并行
  → GPU 几千个核同时上，实际时间可能比 LSTM 还短
```

所以实践中的结论是：**序列不太长时，Attention 反而更快**。这也是为什么 Transformer 能训练几千个 token 的序列，而 LSTM 在几百个 token 后就开始吃力。

## Q3 ✅ 直觉非常接近，帮你精确化

你说"向量描绘的空间的相对距离"——这个直觉很好，但距离（比如欧氏距离）在这里不是最常用的。

Transformer 用的是**点积（Dot Product）**：

$$\text{score}(i, j) = \vec{v}_i \cdot \vec{v}_j = \sum_k v_{ik} \cdot v_{jk}$$

为什么点积比距离更好用？

```text
直觉：
  两个向量方向越接近（夹角越小），点积越大
  两个向量方向相反，点积为负
  两个向量垂直，点积为 0

这正好对应"相关性"的含义：
  方向接近 → 语义相似 → 高注意力
  方向相反 → 语义相反 → 低注意力（甚至负）
  垂直 → 无关 → 注意力为 0
```

你说的"空间距离"和"向量夹角"其实是同一件事的两种描述，都在衡量相似性。Transformer 选择了点积，因为它**计算简单、可以用矩阵乘法批量完成**。
