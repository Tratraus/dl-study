## 参数量表格验收

公式框架正确，但有一个小问题需要确认——**LoRA 的计算少算了一个因子**。

先看 Adapter，你的公式：

```
Adapter (b=64)：2 × (2 × 320 × 64 + 320 + 64)
              = 2 × (40,960 + 384)
              = 2 × 41,344
              = 82,688  ✅

Adapter (b=256)：2 × (2 × 320 × 256 + 320 + 256)
               = 2 × (163,840 + 576)
               = 2 × 164,416
               = 328,832  ✅
```

再看 LoRA，你的公式：

```
你写的：2 × 8 × (320 + 320)

但 LoRA 替换的是 Q 和 V 两个矩阵
每层有 Q 和 V 各一个，所以还要再乘以 2

正确公式：
  LoRA (r=8)：2层 × 2矩阵(Q+V) × r × (d_in + d_out)
            = 2 × 2 × 8 × (320 + 320)
            = 4 × 8 × 640
            = 20,480

  LoRA (r=32)：2 × 2 × 32 × (320 + 320)
             = 4 × 32 × 640
             = 81,920
```

完整对比表：

| 策略 | 可训练参数量 | 相比解冻2层减少 |
|------|------------|--------------|
| 全量微调 | 8,000,000 | — |
| 解冻后2层 | 2,466,883 | 基准 |
| Adapter b=64 | 82,688 | **约 29.8×** ✅ |
| Adapter b=256 | 328,832 | 约 7.5× |
| LoRA r=8 | 20,480 | **约 120×** |
| LoRA r=32 | 81,920 | 约 30× |

LoRA r=8 只用了 **2万参数**，是解冻2层的 1/120，这就是它在大模型时代流行的原因。

## Q2 验收——方向对，但比喻不够精准

你说"大矩阵可以通过两个长方形矩阵相乘得到"，这描述的是**矩阵分解的可行性**，但没有说清楚**为什么这个近似是合理的**。

核心在于"低秩"这两个字：

```
一个随机的 320×320 矩阵，秩最高是 320
它需要完整的 320×320 = 102,400 个参数来描述

但 LoRA 的假设是：
  微调时的权重变化 ΔW 不是随机的
  它只在少数几个"方向"上有变化
  这些方向的数量（秩）远小于 320

更直觉的比喻：
  想象你在学一门新语言
  你不需要重新学习"什么是语言"这件事（高维知识）
  你只需要学习"这门语言特有的几个规则"（低维变化）

  ΔW 就是那"几个规则"
  rank=8 意味着所有变化都可以用 8 个基本方向的组合来描述
```

## Q3 验收——接近正确，但有一个关键细节需要补充

你说"影响原序列各个 token 的 K/V 值"，这个方向是对的，但还差一步：

```
你的理解：前缀 token 影响了原序列 token 的 K/V
                              ↑
实际上方向反了一点

正确的机制：
  前缀 token 被拼接到序列最前面
  它们自己有 K 和 V

  原序列中每个 token 做 Self-Attention 时：
    Q 来自当前 token
    K/V 来自整个序列（包括前缀 token）

  所以前缀 token 的 K/V 被"看到"了
  → 前缀的信息被注入到每个 token 的输出中

类比：
  前缀 token 就像一份"任务说明书"
  放在序列最前面
  每个 token 在处理自己时都会"参考"这份说明书
  从而改变整体的输出行为
```
